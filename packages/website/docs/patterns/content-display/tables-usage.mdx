# Tables usage

*This pattern page is coming soon. It will document table usage best practices and provide ready-to-copy examples.*

## When to use

- **Data display** in structured, scannable formats
- **Sortable data** with column headers
- **Actionable data** with row-level actions
- **Complex data** that benefits from tabular organization

## When not to use

- **Simple lists** (use list components)
- **Card-based layouts** (use card patterns)
- **Form data** (use form patterns)

## Composition

Built from atoms:
- `EuiTable` (container)
- `EuiTableHeader` (column headers)
- `EuiTableBody` (data rows)
- `EuiTableRow` (individual rows)
- `EuiTableHeaderCell` / `EuiTableCell` (cells)

## Variants

- **Basic table** - Simple data display
- **Sortable table** - Column sorting functionality
- **Selectable table** - Row selection with checkboxes
- **Actionable table** - Row-level actions and buttons
- **Responsive table** - Mobile-friendly table patterns
