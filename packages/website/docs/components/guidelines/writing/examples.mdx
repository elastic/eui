## Buttons

Label [buttons](/docs/navigation/button) with their action. Don't use Yes, OK, or Confirm when you can use a verb phrase instead.

<EuiFlexGroup gutterSize="m" alignItems="stretch">
  <Guideline text="Use a verb + noun for a button label." panelStyle={{ minHeight: '292px', display: 'flex', alignItems: 'center' }}>
    <EuiPanel paddingSize="none" style={{ transform: 'scale(0.75)' }}>
      <EuiModalHeader>
        <EuiModalHeaderTitle>Search session complete</EuiModalHeaderTitle>
      </EuiModalHeader>
      <EuiModalBody>
        <EuiText>
          <p>Save your session and return to it later.</p>
        </EuiText>
      </EuiModalBody>

      <EuiModalFooter>
        <EuiButtonEmpty>Manage sessions</EuiButtonEmpty>
        <EuiButton fill>Save session</EuiButton>
      </EuiModalFooter>
    </EuiPanel>
  </Guideline>
  <Guideline text="Use the same verb + noun as in the title." panelStyle={{ minHeight: '292px', display: 'flex', alignItems: 'center' }}>
    <EuiPanel paddingSize="none" style={{ transform: 'scale(0.75)' }}>
      <EuiModalHeader>
        <EuiModalHeaderTitle>Delete this index?</EuiModalHeaderTitle>
      </EuiModalHeader>

      <EuiModalFooter>
        <EuiButtonEmpty color="text">Cancel</EuiButtonEmpty>
        <EuiButton color="danger" fill>
          Delete index
        </EuiButton>
      </EuiModalFooter>
    </EuiPanel>
  </Guideline>
</EuiFlexGroup>

Be sure to read the [full button guidelines](/docs/navigation/button/guidelines).

## Callouts

Use [callouts](/docs/display/callout) to relay informational, success, warning, or error messages related to the content on the page.

<EuiFlexGroup gutterSize="m">
  <Guideline text="Ensure the description provides information in addition to the title" panelStyle={{ minHeight: '186px', display: 'flex', alignItems: 'center' }}>
    <EuiCallOut
      title="Ensure the description provides information in addition to the title."
      iconType="error"
      color="danger"

      You must have the 'create_index' privilege to write data to this index.
    </EuiCallOut>

  </Guideline>

## Empty prompts

The best empty state lets users know what is happening, why it is happening, and what to do about it. Here are four types of frequently used [empty prompts](/docs/display/empty-prompt).

<Guideline text="Prompt first-time users to take action.">
  <EuiEmptyPrompt
    style={{ transform: 'scale(0.75)'}}
    iconType="addDataApp"
    title={<h2>Register your first repository</h2>}
    body={<p>Create a place where your snapshots will live.</p>}
    actions={
      <EuiButton color="primary" fill>
        Register a repository
      </EuiButton>
    }
  />
</Guideline>

<Guideline text="If there is nothing to show, give instructions on what to do.">
  <EuiEmptyPrompt
    style={{ transform: 'scale(0.75)'}}
    iconType="searchProfilerApp"
    title={<h2>No queries to profile</h2>}
    body={
      <p>
        Enter a query, click the Profile button, and see the results
        here.
      </p>
    }
  />
</Guideline>

<Guideline text="Introduce a feature.">
  <EuiEmptyPrompt
    style={{ transform: 'scale(0.75)'}}
    iconType="discoverApp"
    title={<h2>Introducing our table view</h2>}
    body={
      <p>
        Explore your data inside Discover with our improved sorting and
        filtering tools.
      </p>
    }
    actions={
      <EuiButton color="primary" fill>
        Go to Discover
      </EuiButton>
    }
  />
</Guideline>

<Guideline text="If using an empty prompt for an error, explain next steps.">
  <EuiEmptyPrompt
    style={{ transform: 'scale(0.75)'}}
    iconType="lock"
    title={<h2>We couldn&apos;t log you in</h2>}
    body={
      <p>
        Check your credentials and try again. If you still can&apos;t
        log in, contact your system administrator.
      </p>
    }
  />
</Guideline>

Be sure to read the [full empty prompt guidelines](/docs/display/empty-prompt/guidelines).

## Labels

Avoid long [labels](/docs/forms/form-layouts/#form-labels), but don't sacrifice clarity. If needed, put additional information in help text and tooltips.

<EuiFlexGroup gutterSize="m">
  <Guideline text="Use labels that say what the input does." panelStyle={{ minHeight: '155px', display: 'flex', alignItems: 'center' }}>
    <div>
      <EuiFormRow>
        <EuiCheckbox
          onChange={() => {}}
          id={htmlIdGenerator()()}
          label="Combine values in other bucket"
        />
      </EuiFormRow>
      <EuiFormRow label="Bucket label">
        <EuiFieldText />
      </EuiFormRow>
    </div>
  </Guideline>

  <Guideline text="Use specific labels." panelStyle={{ minHeight: '155px', display: 'flex', alignItems: 'center' }}>
    <div>
      <EuiFormRow>
        <EuiCheckbox
          onChange={() => {}}
          id={htmlIdGenerator()()}
          label="Limit incoming traffic"
        />
      </EuiFormRow>
      <EuiFormRow
        label="Source"
        helpText={
          <span>
            Enter CIDR or IP address, for example 192.168.132.6/22.
          </span>
        }
      >
        <EuiFieldText />
      </EuiFormRow>
    </div>
  </Guideline>
</EuiFlexGroup>

## Modals

[Modals](/docs/layout/modal) are typically used for confirmation when the user might lose data.

<EuiFlexGroup gutterSize="m">
  <Guideline text="For the body text, use one to two short sentences that explain the consequences." panelStyle={{ minHeight: '360px', display: 'flex', alignItems: 'center' }}>
    <EuiPanel paddingSize="none" style={{ transform: 'scale(0.75)' }}>
      <EuiModalHeader>
        <EuiModalHeaderTitle>
          <>New dashboard already in progress</>
        </EuiModalHeaderTitle>
      </EuiModalHeader>
      <EuiModalBody>
        <EuiText>
          <p>You can continue editing or start with a blank dashboard.</p>
        </EuiText>
      </EuiModalBody>

      <EuiModalFooter>
        <EuiButtonEmpty>Continue editing</EuiButtonEmpty>

        <EuiButton fill>Start over</EuiButton>
      </EuiModalFooter>
    </EuiPanel>
  </Guideline>
  <Guideline text="Some cases require three buttons." panelStyle={{ minHeight: '360px', display: 'flex', alignItems: 'center' }}>
    <EuiPanel paddingSize="none" style={{ transform: 'scale(0.75)' }}>
      <EuiModalHeader>
        <EuiModalHeaderTitle>
          <>You have unsaved changes</>
        </EuiModalHeaderTitle>
      </EuiModalHeader>
      <EuiModalBody>
        <EuiText>
          <p>
            When you exit editing mode, you can save or discard your
            changes. You can&apos;t recover discarded changes.
          </p>

      <EuiModalFooter style={{ flexWrap: 'wrap' }}>

      </EuiModalFooter>
    </EuiPanel>
  </Guideline>
</EuiFlexGroup>

<EuiFlexGroup gutterSize="m">

## Text fields

Place hints and instruction outside a [text field](/docs/forms/form-controls/#text-field) so it is always visible to the user.

<EuiFlexGroup gutterSize="m">
  <Guideline text="Help users make the right decision by clarifying what goes inside a field.">
    <EuiFormRow
      label="Set a custom interval"
      helpText={
        <span>Use shorthand notation, such as 30s, 10m, or 1h.</span>
      }
    >
      <EuiFieldText />
    </EuiFormRow>
  </Guideline>
  <Guideline text="Use complete sentences and ending punctuation.">
    <EuiFormRow
      label="Name"
      helpText={<span>Your name will be on public display.</span>}
    >
      <EuiFieldText name="text" />
    </EuiFormRow>
  </Guideline>
</EuiFlexGroup>

Use clear language for [validation](/docs/forms/form-validation) messages.

<EuiFlexGroup gutterSize="m">
  <Guideline text="Prefer this format for a required field.">
    <EuiFormRow label="Name" isInvalid={true} error="Name is required.">
      <EuiFieldText name="text" isInvalid={true} />
    </EuiFormRow>
  </Guideline>
  <Guideline text="Tell users what happened and how to fix it.">
    <EuiFormRow
      label="Address"
      error="Choose a different address as this one already exists."
      isInvalid={true}
    >
      <EuiFieldText
        value="1234 Elastic Ave"
        onChange={() => {}}
        isInvalid={true}
      />
    </EuiFormRow>
  </Guideline>
</EuiFlexGroup>

## Toasts

A common use of [toasts](/docs/display/toast) is as a success message.

<EuiFlexGroup gutterSize="m">
  <Guideline text="For common actions such as create, add, delete, remove, and save, include the object type, the object name if available, and the past tense of the action. Don't include &quot;successfully&quot;. It's implied.">
    <EuiToast
      color="success"
      style={{ maxWidth: 300 }}
      title="User 'Casey Smith' was added"
    />
  </Guideline>
  <Guideline text="For a message about multiple objects, include the object count, but not the names of the objects.">
    <EuiToast
      color="success"
      style={{ maxWidth: 300 }}
      title="4 visualizations were deleted"
    />
  </Guideline>
</EuiFlexGroup>

Be sure to read the [full toast guidelines](/docs/display/toast/guidelines).
