# Page layout

*This pattern page is coming soon. It will document page layout best practices and provide ready-to-copy examples.*

## When to use

- **Application pages** with consistent structure
- **Content layouts** with headers, navigation, and content areas
- **Dashboard layouts** with multiple content sections

## When not to use

- **Simple content** (use individual layout components)
- **Modal content** (use modal patterns)

## Composition

Built from atoms:
- `EuiPage` (container)
- `EuiPageHeader` (page header)
- `EuiPageSideBar` (navigation)
- `EuiPageBody` (content area)
- `EuiPageContent` (content sections)

## Variants

- **Basic page** - Simple header and content
- **Dashboard page** - Multiple content sections
- **Detail page** - Header with actions and content
- **Wizard page** - Multi-step page layout
